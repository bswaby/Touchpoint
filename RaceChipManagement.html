Step 1: Post-Race Organization

‚è∞ Timeline: Complete within 48 hours of race completion for highest fee collection success

Tasks:

  1. Clean all chips using wet rags
  2. Organize chips back into their designated containers
  3. Document missing chips:
    • Write down all missing chip numbers
    • Cross-reference missing chips with team assignments
    • Send missing chip list to coach if you're not completing remaining steps

Organization Tips:

  • Stack chips in groups of 10
  • Arrange stacks in rows of 100 for easy counting
  • Use DeWalt cases with 25-chip stacks for quick visual inventory
Note: If you're also handling chip verification, consider completing it immediately after organization to ensure thoroughness.
‚ö†Ô∏è Important: Extra chips that are used at the check-in table don't always make it back. Make sure to ask about these before you leave the race.

Step 2: Chip Verification

Purpose: Ensures maximum chip read success for future races. Requires attention to detail and persistence.

Setup:

  1. Connect Trident USB Registration Reader to computer
  2. Open Trident TinyScore software

Verification Process:

1. Configure Reader:

  • Navigate to Reader Setup tab
  • Verify reader detection
  • If not detected, try changing COM port settings

2. Test Each Chip:

  • Click "Live Tag Data" to begin verification
  • Scan all tags systematically
  • Verify displayed number matches physical tag number

3. Document Issues:

  • Missing chips: Note numbers and correspond with team assignments for coach
  • Faulty reads: Dispose of defective chips
Troubleshooting: If reader performance degrades, unplug USB and reconnect (rare occurrence: ~2 times in 30+ sessions)
‚ö†Ô∏è Watch for Duplicates: Coaches/teams will turn in chips from previous races causing duplicates from ones you have previously programmed. This is another critical reason why it's important to verify ALL chips during this step.

Step 3: Replacement/New Chip Programming

Preparation:

  1. Label replacement chips with permanent marker
  2. Close Trident TinyScore
  3. Open Trident TagScan

Programming Process:

1. Scan New Chips:

  • Click "Start Logging Tags"
  • Scan chips individually
  • Verify each chip appears in the list
  • Click "Save Tags"
  • Save file as: NewTagsYYYYMMDD.txt

2. Update Chip Numbers:

  • Open saved file from desktop folder
  • File format: number,hexadecimal (e.g., 1,0580a043a899)
  • Renumber each entry to match missing chip numbers
  • Example: Change 1,0580a043a899 to 31,0580a043a899 for missing chip #31
Tip: Use Excel for bulk renumbering with auto-numbering feature

3. Update Master File:

‚ö†Ô∏è CAUTION: This is the master chip database - handle carefully
  • Open latest master file: YYYYMMDD - TagsMP.txt
  • Update entries with new chip data
  • Save as new file using same naming format: YYYYMMDD - TagsMP.txt

4. Import to System:

  • Close Trident TagScan
  • Open Trident TinyScore
  • In Reader Setup tab: Click "Clear tag reads database"
  • In Tag Registration tab:
    • Click "Delete All Tag Registrations"
    • Confirm deletion
    • Click "Import" and select updated master file
  • Verify Bib No. 1 appears correctly
Important: Remove "Num, Tag" headers if present (required for FinishLynx compatibility)

5. Final Verification:

üî¥ CRITICAL: Repeat Step 2 verification process with all chips

Step 4: Finish Work

1. Documentation:

  • Compile final missing chip list with team cross-references
  • Send to coach for fee collection

2. Coordination:

  • Arrange chip delivery to appropriate personnel
  • Ensure chips are ready for packet stuffing

üí° Tips & Tricks

Cleaning Tips:

Remove permanent marker: Spray sunblock effectively removes permanent marker from chips

Storage Organization:

  • DeWalt cases hold chips in 25-unit stacks
  • This configuration makes missing chips immediately visible

Best Practices:

  • Complete all steps in sequence when possible
  • Double-check all work before passing to next person
  • Maintain detailed records of all missing or faulty chips
  • Always backup master files before making changes
